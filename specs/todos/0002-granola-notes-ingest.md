# Granola notes ingest

Status: superseded by `specs/studio/MEETINGS.md`.

The Granola ingest idea is now part of the Studio target design. `MEETINGS.md` treats meeting
memory as the product profile and Granola as an import and bridge source. The older standalone
ingest plan was temporary ideation and should not be used as the contract for new work.

Use the Studio spec for:

- source, meeting, span, annotation, fact, relation, and extraction records;
- Granola API, local-cache, CSV, and MCP source roles;
- import mapping, bridge semantics, identity handling, and fidelity reports;
- privacy, consent, retention, surfaces, and cross-profile promotion;
- implementation sequence and unfinished tasks.
